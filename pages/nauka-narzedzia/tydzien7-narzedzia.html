Node.js

Instalacja dla Windowsa

Instalujemy Node.js ze strony: nodejs.org

Dla ubuntu

Żeby zainstalować node.js trzeba postępować w następujący sposób:

sudo apt-get update
sudo apt-get install nodejs

Żeby sprawdzić czy instalacja się powiodła możemy sprawdzić wersję zainstalowaną:

nodejs -v

node to taki JavaScript uruchamiany poza przeglądarką. Node.js nie ma dostępu do DOM i do wszystkich api związanych z przeglądarką. Nie ma query.selector'ów

Pierwszy program🙂

script.js

const name = 'Paweł';
console.log(`${name}`);
console.log('hello node!');

Żeby ten skrypt uruchomić w nodzie musimy wpisać komendę:

node script.js

W nodzie można pisać tzw. backend. Czyli javaScript'owi developerzy piszą całe aplikacje serwerowe i to działa :)

NPM i NPX

NPM

Nie jest to tłumaczone jako node package manager chociaż nim jest.

Czyli npm to menadżer paczek do noda. Zawiera on repozytorium, z którego możemy pobrać programy pozwalające na usprawnienie nam pracy. Dzięki nim łątwiej możemy budować nasz projekt.

Po co nam te paczki?
  • dodajemy do projektów sporo gotowych bibliotek / narzędzi
  • nie można polegać na konfiguracji edytora / jego wtyczkach
  • uniwersalne źródło ze wszystkimi narzędziami
  • dostęp do aktualizacji

npm to taki app store / sklep play dla frontendowca
npm to taka skrzynka z narzędziami, których możemy użyć w naszym projekcie

No to zaczynamy😛

Żeby zainstalować npm w ubuntu odpalamy:

sudo apt install npm

Żeby odpalić npm'a dla danego projektu wpisujemy:

npm init

Polecenie te wykonujemy tylko raz w projekcie i robimy je w celu uzyskania dostępu do repozytoriów npm'a. Utworzy on nam plik package.json

Jeśli wpisemy npm init -y to utworzy również plik package.json , ale nie zada nam pytań o dane naszego projektu. Które wchodzą w skład naszego pliku

W package.json znajdują się informacje o naszym projekcie. Czyli jego opis, wersja projektu, opis projektu, stosowane paczki itd. Trzyma również informacje o tym jakie paczki są nam potrzebne przy budowaniu naszego projektu.

W package-lock.json znajdują się informacje o zainstalowanych paczkach w projekcie i o wersji tych paczek. Te informacje są ważna o tyle, żeby nam się projekt nie rozjeżdżał

Instalacja pakietów
  • npm install --save - używamy tej opcji jeśli coś ma być później widoczne w przeglądarce😛
    • --save-dev - używamy tej opcji jeśli instalujemy narzędzie, które będziemy używać tylko do rozwoju naszego projektu np. serwer lokalny
  • npm install pozwala na ponowne zainstalowanie tych samych paczek zdefiniowanych w package.json (np. gdy nie mamy katalogu node_modules w którym są one przechowywane)
Katalog node_modules

Katalog, w którym przechowywane są wszystkie nasze paczki.

Nie commitujemy katalogu node_modules. Każdy może zrobić npm install i ściągnąć sobie wymagane paczki). Czyli dodajemy go do pliku .gitignore

NPM Scripts

NPM skrypty są te komendy, które możemy sobie zdefiniować w pliku package.json, który jest w naszym projekcie. Żeby dodać taki skrypt, to w sekcji scripts definiujemy nazwę skryptu i polecenie, które ma wykonać.  Na początek tworzymy skrypt:

"start": "node script.js",

Żeby odpalić skrypt w konsoli wpisujemy:

npm start

Dla skryptów opalanych npx (np. dla cowsay 😛) skrypt zapisujemy podobnie :

"cow": "cowsay hello world!",

ale odpalamy je poleceniem:

npm run cow

SASS / SCSS

Problemy z CSS

  • porozrzucane klasy (bałagan w plikach)
  • @media oddzielone od elementu (konieczność ponownego definiowania)
  • kiepska czytelność plików (szczególnie przy zagnieżdżeniach)
  • brak możliwości definiowania funkcji do robienia podobnych rzeczy

Dużą ilość tych problemów rozwiązują preprocesory. Jednym z popularniejszych preprocesorów jest SASS ( Sass documentation )

SASS potrzebuje narzędzia do działania. Na tym etapie nie będziemy instalować tego narzędzia. Na początku skupimy się na zasadach działania tego narzędzia i jak go używać.

SASS vs SCSS

SASS jest narzędziem, ma on swoją składnię i dodatkowo ma składnię SCSS.

Działanie preprocesorów polega na tym, że na początku piszemy style w pliku .scss, a następnie program SASS zamienia to na plik .css który jest zrozumiały dla przeglądarki. \

Sprawdźmy jak działa SASS

https://www.sassmeister.com/ na tej stronie będziemy pisać pierwszy swój kod scss 😛 Od razu będziemy mieli widok jak to będzie wyglądało w kodzie css po użyciu Sass

Wszystko co zapisalibyśmy w css, zadziała również w Sass'ie

zmienne

W SCSS'ie występują zmienne i żeby je zdefiniować używamy znaczka $. Zmienne te nie będą się przepisywać do CSS, tylko będą wykorzystywane w SCSS.

$myFavouriteGrey: #ccc;

Wszędzie, gdzie dana zmienna będzie wykorzystana w kodzie, to będzie automatycznie uzupełniana jej wartość w kodzie css. Np.

color: $myFavouriteGrey;
@mixin

Jest to funkcja w której możemy zdefiniować kilka własności i później móc je szybko zastosować w kodzie.

@mixin borderBackground {
border-radius: 10px;
background: hotpink;
}

Żeby wywołać własności z @mixin'a przywołujemy go poprzez include

include borderBackground

Możemy również Mixin'em definiować parametry:

@mixin borderBackgroundParameter($radius, $background) {
border-radius: $radius;
background: $background;
}

@include borderBackgroundParameter(20px, yellow)
@extend

Pozwala na zaimportowanie stylów użytych przy innym selektorze.

@extend .font-style;

Nasz plik scss'owy wygląda następująco:

$myFavouriteGrey: #ccc;
$tablet: 768px;
$desktop: 1024px;

@mixin borderBackground {
border-radius: 10px;
background: hotpink;
}

@mixin borderBackgroundParameter($radius, $background) {
border-radius: $radius;
background: $background;
}

.font-style {
font-size: 13px;
color: green;
}

.main {
color: $myFavouriteGrey;
@include borderBackground;
@extend .font-style;
}

.main::before {
fron-size: 20px;
content: 'hi';
}

@media (min-width: $desktop) {
.main {
color: red;
@include borderBackgroundParameter(20px, yellow);
}
}

dla porównania plik css będzie wyglądał następująco

.font-style, .main {
font-size: 13px;
color: green;
}

.main {
color: #ccc;
border-radius: 10px;
background: hotpink;
}

.main::before {
fron-size: 20px;
content: "hi";
}

@media (min-width: 1024px) {
.main {
color: red;
border-radius: 20px;
background: yellow;
}
}
& parent selector

Pozwala na zapisanie jednej grupy BEM'owej w jednym miejscu. Poprzez zagnieżdżanie własności, mamy wszystko w jednym miejscu, a preprocesor zamieni to na czytelny kod dla przeglądarki.

SCSS

.about-sass {
color: red;
&__title {
font-size: 20px;
&--smaller {
font-size: 16px;
}
}
@media (max-width: 768px) {
.about__title {
font-size: 18px;
}
}

CSS

.about-sass {
color: red;
}
.about-sass__title {
font-size: 20px;
}
.about-sass__title--smaller {
font-size: 16px;
}
@media (max-width: 768px) {
.about-sass .about__title {
font-size: 18px;
}
}

Kiedy korzystamy z preprocesorów nie edytujemy plików wynikowych .css

node-sass

Instalujemy paczkę node-sass , która pozwoli nam z skorzystać z SCSS w naszym projekcie homepage.

npm install node-sass

Żeby node-sass podmienił utworzony plik scss na plik css wpisujemy polecenie:

npx node-sass scss/main.scss css/main.css

Istnieje również funkcja śledzenia zmian w pliku scss i automatycznie nadpisywanie pliku css. jest to możliwe poprzez dodanie flagi -w (watch) do kodu node-sass.

npx node-sass -w scss/main.scss css/main.css
Skrypt node-sass

Dodajemy skrypt do package.json, żeby ułatwić sobie i przyspieszyć pracę.

"sass": "npx node-sass -w scss/main.scss css/main.css",

Żeby wykonać skrypt w terminalu wpisujemy:

npm run sass
