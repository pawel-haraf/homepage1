Podstawowe informacje o CSS'ie

CSS (czyli Cascading Style Sheets - Kaskadowe arkusze stylów) odpowiada za prezentacje dokumentu HTML. Czyli za to jakie są używane czcionki, kolor tła, wielkość marginesów i obramowania itp. CSS wpływają na wygląd elementów HTML na ekranie, wydrukach itp.

Budowa reguły CSS

Składnia CSS to reguły i wygląda ona następująco:

selektor { właściwość: wartość; }

Za pomocą selektora wybieramy co chcemy ostylować. Pliki zawierające kod CSS mają rozszerzenie *.css. Jest możliwość korzystania z reguł CSS bez używania zewnętrznych plików.

Czas na pierwsze stylowanie

Znacznik <style> Umieszczamy go w sekcji <head>. Dlatego, że to co tam wpisujemy, nie jest widoczne w przeglądarce. Definiujemy te wszystkie rzeczy, które wpływają na naszą stronę, ale nie wpływają na jej treść. Wpływają na jej wygląd i zachowanie. W środku wpisujemy interesujący nas selektor, dzięki któremu zmieniamy wygląd interesującego nas elementu np. nagłówka <h1>. Czyli możemy to zapisać w następujący sposób:

<style>
body {
background: red; }
h1 {
color: blue; }
p {
color: gray; }
h2 {
color: #111111; }
</style>

Kolor czcionki dowolnego elementu możemy określać poprzez własność color i jako wartość zapisać go jako angielskie słowo np. blue, albo za pomocą kodu szesnastkowego np. #111111 (co w tym przypadku oznacza czarny). Kolejne dwójki w tym zapisie oznaczają kanał czerwony, kanał zielony i kanał niebieski.

Innym sposobem na dołączenie stylów do dokumentu jest umieszczenie w osobnym pliku (zewnętrzy arkusz stylów) i dodanie poprzez: Znacznik<link>:

<link href="style.css" rel="stylesheet" type="text/css">

Umożliwia, to zwiększenie czytelności kodu. Szczególnie gdy konfigurujemy wygląd większej ilości elementów. Zapobiega to rozrastaniu się dokumentu do dużych wielkości, poprzez kolejne linijki kodu zawierające własności CSS. Ciężko, będzie dobrać się do treści, a sam dokument będzie mniej czytelny.

Możliwa jest również zmiana tła danego elementu np. poprzez użycie właściwości background. Kolor tła zapisujemy analogicznie jak to było w przypadku koloru czcionki.

Podsumowanie:

Style CSS:

  • Odpowiadają za prezentację treści strony
  • Nie mają jednej wersji (rozwój modułów), potocznie mówi się, że aktualna wersja to 3
  • Przeglądarki potrafią różnie interpretować CSS co prowadzi do tego, że…
  • … pisanie stylów powoduje wiele frustracji u programistów (ale jest też niezłym wyzwaniem :))
  • Całą specyfikację znajdziesz na stronie w3c: https://www.w3.org/TR/CSS/
