WTF: Klan czyli DOM jako rodzina elementów i dziedziczenie własności CSS

DOM ma strukturę podobną do drzewa genealogicznego. Czyli jest główny rodzic <html>, posiada swoje dzieci <head> i <body>. One z kolei posiadają swoje dzieci, które mają swoje. Tworzy się z tego spora familia. 😛

Dziedziczenie

Wszystkie dzieci dostają atrybuty swojego rodzica, pod warunkiem, że nie mają już zdefiniowanych tych atrybutów.

Jeśli rodzice i dzieci nie mają zdefiniowanych żadnych atrybutów, to przeglądarka wykorzystuje te style, które ma domyślnie zdefiniowane.

własność: inherit

Określa, że dana własność ma być dziedziczona z rodzica. Oznacza że jeśli dla koloru linków przypiszemy inherit to kolor takiego złącza będzie taki jak rodzica (np. nagłówka lub paragrafu)

Normalizacja vs reset

Reset

Polega na wyłączeniu wszystkich domyślnych stylów przeglądarki. Jest to technika, polegająca zapewnieniu sobie pracy na czystej kartce. Przez to bez określenia swoich własności CSS, strona traci na przejrzystości. Ciężko odróżnić elementy od siebie. "Typowy" reset to:

* {
margin:0;
padding: 0;
outline: none; /* Ta własność w dużym stopniu ogranicza dostępność dokumentu. Tracimy możliwość orientowania się co jest co w dokumencie. przy naciskaniu klawiasza TAB nie jest oznaczane miejsce, gdzie się aktualnie znajdujemy */
}

ul, ol {
list-style: none;
}
Normalizacja

Normalizacja ma na celu ujednolicenie wyglądu stylów domyślnych w różnych przeglądarkach (różni producenci różnie je zdefiniowali w swoich produktach). W tym powstał projekt który rozwiązuje te problemy i sprawia, że nawet te rzeczy, których nie ostylujemy będą wyglądać podobnie w różnych przeglądarkach. W tym celu powstał specjalny plik CSS'owy normalize.css który możemy dodać do dokumentu html.

Miksujemy!

Żeby nie było za łatwo, to możemy połączyć reset z normalizacją😛

Czyli stosujemy własności CSS z pliku normalize.css oraz dodatkowo do pliku, ze swoimi stylami dopisujemy:

html {
box-sizing: border-box;
}

*, *:before, *:after {
box-sizing: inherit;
}

Flexbox -Wyśrodkujmy coś

display: flex

Flex jest to własność, która zamienia nam domyślny display: block na nowy system pozycjonowania i rozmieszczania elementów. Flex pracuje w dwóch osiach: osi poziomej i pionowej.

  • flex-direction - określa kierunek w jakim będą ustawiane kolejne elementy. Jego wartości to:
    • row (ustawia elementy obok siebie)
    • column (ustawia elementy pod sobą)
    • row-reverse (ustawia elementy obok siebie - od prawej do lewej)
    • column-reverse (ustawia elementy pod sobą - od dołu do góry)
  • flex-wrap - zawijanie wiersza/kolumny:
    • nowrap - nie zawijaj (stara się zmieścić wszystko w jednej linii/kolumnie)
    • wrap - zawija wiersze/kolumny ( jeśli element osiągnie swoją wielkość i reszta elementów się nie mieści zostaną przerzucone do następnej linii/kolumny)
    • wrap-reverse- zawija wiersze kolumny, umieszczając pierwszy box w prawym dolnym rogu
  • flex-flow skrócona wartość, która umożliwia nam podanie własności flex-direction i flex-wrap w jednym miejscu, np.:
  • flex-flow: row nowrap;
  • justify-content - ustawianie elementów wzdłuż osi flex-direction
    • flex-start domyślna wartość - przykleja elementy do lewej/górnej krawędzi
    • flex-end przykleja elementy do prawej/dolnej krawędzi
    • center ustawia elementy na środku
    • space-between rozstawia elementy na całą szerokość ekranu (przyklejając elementy do prawej i lewej krawędzi), a między poszczególnymi elementami zostawia równy dystans (margines)
    • space-around rozstawia elementy na całą szerokość ekranu, ustawia równe wolne przestrzenie (marginesy) pomiędzy elementami oraz od krawędzi
  • justify-content - ustawianie elementów wzdłuż osi flex-direction
    • stretch domyślna wartość- rozciąga elementy
    • flex-start
    • flex-end
    • center
    • space-between
    • space-around
  • align-content ustawianie wierszy/kolumn wzdłuż osi przeciwnej do flex-direction (taki justify-content dla drugiej osi)
    • stretch domyślna wartość- rozciąga elementy
    • flex-start
    • flex-end
    • center
    • space-between
    • space-around
  • align-items ustawianie elementów wzdłuż osi przeciwnej do flex-direction w obrębie pojedynczego wiersza/kolumny (jeśli są różnej wielkości elementy w jednej linii, to wyrównuje określony sposób)
    • stretch domyślna wartość- rozciąga elementy
    • flex-start
    • flex-end
    • center
    • space-between
    • space-around
    • baseline ustawia element względem dolnej krawędzi pierwszego wiersza
  • align-self ustawianie elementu wzdłuż osi przeciwnej do flex-direction. Pozycjonujemy pojedyńczy element.
    • stretch domyślna wartość- rozciąga elementy
    • flex-start
    • flex-end
    • center
    • space-between
    • space-around
    • baseline ustawia element względem dolnej krawędzi pierwszego wiersza
  • order kolejność, ta własność umożliwia ustalenie kolejności rozmieszczenia elementów, niezależnie od tej domyślnej. Elementy o niższych wartościach przyporządkowanej liczby będą wyświetlane jako pierwsze i kolejno z coraz wyższymi indexami.
    • liczba całkowita (domyślnie 0)
  • flex-grow współczynnik odpowiadający za to jak bardzo nasz element powinien rosnąć jeśli w wierszu, kolumnie zostaje jakieś wolne miejsce.
    • liczba całkowita (domyślnie 0)
  • flex-shrink własność sprawia że dany element kurczy się tylko do osiągnięcia wielkości minimalne i później już się nie zmienia
  • flex-basis rozmiar bazowy - jeśli element nie ma określonej wielkości przyjmie on rozmiar bazowy
  • flex: <flex-grow> <flex-shrink> <flex-basis></flex-basis> skrót umożliwiający nam zapisanie kilku własności w jednej linijce
    • liczba całkowita (domyślnie 1)

Plac zabaw z flex'em

CSS: Tła

display: flex

Własności w CSS'ie związane z tłami zaczynają się od background , funkcjonuje to jako osobna własność ( skrócona ), która zawiera w sobie bardziej rozwinięte własności związanych z tłami. Składa się ona z:

  • background-image: url('sciezka-do-pliku') - wstawia obrazek jako tło
  • background-position: ustawia nam tło względem elementu, które to tło dostaje
    • 0 0 (domyślna wartość górny lewy róg)
    • top right
    • bottom left
    • center center
    • xx% xx%
    • xxpx xxpx
    • … …
  • background-repeat:powtarzanie tła
    • repeat (domyśłna wartość)
    • repeat-x - powtarzane tylko w osi x
    • repeat-y - powtarzane tylko w osi y
    • no-repeat - nie powtarzane
    • round - otacza nasz element
    • space - nie przycina obrazka, tylko rozmieszcza go równomiernie elemencie
  • background-attachment: sposób zachowania się tła w momencie przesuwania treści
    • scroll - przesuwa tło razem ze stroną (treścią)
    • fixed - tło nie rusza się, jest nieruchome
    • local - tło przesuwa się razem z treścią elementu (jeśli dla elementu nadana jest wysokość oraz overflow: scroll )
  • background-origin: definiuje nam w którym miejscu nasze tło powinno się zaczynać
    • content-box
    • padding-box
    • border-box
  • background-clip: pokazuje w którym miejscu tło powinno być cięte
    • padding-box
    • border-box
    • content-box
  • background-color: wartość koloru;
    • rgba(255,255,255,0.4) tło może być pół przezroczyste, czwarta wartość określa stopień przezroczystości
  • background-size: określa jakie duże ma być nasze tło
    • auto
    • cover - obrazek będzie wypełniać całe okno nie zależnie od rozmiaru okna
    • contain - będzie starał się wypełnić ekran w miarę jak będziemy zmniejszać/zwiększać wielkość okna i starając się umieścić całe zdjęcie
    • <width> <height>
  • gradienty - fajną stroną do robienia gradientów jest:
    gradient-backgrounds
  • background:
    • url(sweettexture.jpg) /* image */
    • top center / 200px 200px /* position / size */
    • no-repeat /* repeat */
    • fixed /* attachment */
    • padding-box /* origin */
    • content-box /* clip */
    • red ; /* color */
