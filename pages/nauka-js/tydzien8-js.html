CommonJS vs ES6 Modules

My będziemy najczęściej korzystać z ES6 Modules.

CommonJS najczęściej wykorzystywane w node'zie.

Żeby móc korzystać z dobroci modułów musimy do naszego znacznika script dopisać atrybut type=module

Eksport nazwany

W kodzie JavaScript musimy zaznaczyć co chcemy wyeksportować (funkcję, zmienną). Robimy to poprzez dopisanie export przed naszym fragmentem kodu przeznaczonym do eksportu np.

export const hello = (name) => {
return `Cześć ${name}`;
}

export const korsi = 'Maciek'

Następnie zaznaczamy po słowie import co chcemy zaimportować do naszego kodu:

import {hello, korsi} from './hello.js';
console.log(hello(korsi)); import {pierwszy, drugi} from 'nazwa-modułu' import {pierwszy, drugi} from 'ścieżka/do/pliku'

Jeśli chcemy zmienić nazwę w imporcie nazwanym, to stosujemy składnie:

import {bardzoDługaNazwa as pierwszy} from ...
Eksport domyślny

Działa w taki sposób, że nie musimy na sztywno definiować nazwy rzeczy, która chcemy zaimportować. Wystarczy mieć domyślną nazwę, która umożliwi zaimportowanie odpowiedniego skryptu.

import human from './maciek';

console.log(human); import nazwaRzeczy from 'nazwa-modułu'

import nazwaRzeczy from 'ścieżka/do/pliku

import * as newName from ... Pozwala nam zaimportowanie wszystkich importów z danego pliku . Importuje je jako obiekt pod jedną zmienną.
