Po co ten grid?

Potrzeba stworzenia prostego systemu do budowania layoutów (tabele, floaty, div-oza itp. nie spełniały oczekiwań i były kiepską praktyką).

Przyjazny zapis i możliwość łatwej edycji w kontekście responsywności. Standard dopasowany do aktualnego stanu stron internetowych

  • display: grid własność grid działa tylko na swoje dzieci i ustawia elementy w siatce
    • grid-template-columns: 100px 300px; określamy wielkość i ilość kolumn ( w tym przypadku powstaną dwie kolumny o szerokości 100 i 300px
    • grid-template-rows: 100px 300px; własność ta definiuje nam liczbę wierszy i jaką będą miały szerokość
    • fr nowa jednostka, jest to w pewnym sensie ułamek, oraz jest brana z wolnej przestrzeni nie zdefiniowanej przez jednostki. Czyli jeśli mamy trzy kolumny o szerokościach 1fr 1fr 2fr, to pierwsza i druga kolumna będą miały szerokość 1/4 szerokości ekranu, a trzecia będzie miała 2/4 szerokości ekranu (1+1+2=4).  Oraz gdy mamy dwie kolumny 200px 1fr; to 1fr będzie wynosić dokładnie tyle co wolna przestrzeń jaka została po zajęciu 200px przez 1szą kolumnę
    • repeat(ileRazy, wartość) określa ile chcemy mieć kolumn/wierszy o takiej samej szerokości
    • column-gap własność która tworzy przerwy między kolumnami
    • row-gap własność która tworzy przerwy między wierszami
    • minmax(minWartość, maxWartość)

Media queries & mobile-first

Media queries umożliwia na dostosowanie  strony do wyświetlania na różnych rodzajach urządzeń (np. komputery, tablety, komórki itp.).

  • @media
    • media type pozwala na określenie dla jakich urządzeń są nasze css
      • screen
      • print
      • all
      • ...
    • media feature
      • min-width
      • max-width
      • color
      • aspect-ratio
      • orientation
      • ...
    • możemy też łączyć te ustawienia za pomocą:
      • and
      • not - zaprzeczenie
      • "," - działa jak or (lub) w JS'ie

Zapis @media możemy potraktować jako nowy css dla podanych warunków, zostały zdefiniowane dla danego przypadku.

Media queries dobrze definiować po zdefiniowaniu domyślnego wyglądu naszego elementu.

mobile-first

Podejście polega na tym, że najpierw stylujemy stronę dla urządzeń mobilnych,  a później za pomocą media queries  ustalamy elementy jak one mają wyglądać na większych rozdzielczościach.

break-point

Są to zdefiniowane punkty, w których następuje zmiana właściwości css (zdefiniowane w media queries)
