Podstawowe informacje o CSS'ie

CSS (czyli Cascading Style Sheets - Kaskadowe arkusze stylów) odpowiada za prezentacje dokumentu HTML. Czyli za to jakie są używane czcionki, kolor tła, wielkość marginesów i obramowania itp. CSS wpływają na wygląd elementów HTML na ekranie, wydrukach itp.

Budowa reguły CSS

Składnia CSS to reguły i wygląda ona następująco:

selektor { właściwość: wartość; }

Za pomocą selektora wybieramy co chcemy ostylować. Pliki zawierające kod CSS mają rozszerzenie *.css. Jest możliwość korzystania z reguł CSS bez używania zewnętrznych plików.

Czas na pierwsze stylowanie

Znacznik <style> Umieszczamy go w sekcji <head>. Dlatego, że to co tam wpisujemy, nie jest widoczne w przeglądarce. Definiujemy te wszystkie rzeczy, które wpływają na naszą stronę, ale nie wpływają na jej treść. Wpływają na jej wygląd i zachowanie. W środku wpisujemy interesujący nas selektor, dzięki któremu zmieniamy wygląd interesującego nas elementu np. nagłówka <h1>. Czyli możemy to zapisać w następujący sposób:

<style>
body {
background: red; }
h1 {
color: blue; }
p {
color: gray; }
h2 {
color: #111111; }
</style>

Kolor czcionki dowolnego elementu możemy określać poprzez własność color i jako wartość zapisać go jako angielskie słowo np. blue, albo za pomocą kodu szesnastkowego np. #111111 (co w tym przypadku oznacza czarny). Kolejne dwójki w tym zapisie oznaczają kanał czerwony, kanał zielony i kanał niebieski.

Innym sposobem na dołączenie stylów do dokumentu jest umieszczenie w osobnym pliku (zewnętrzy arkusz stylów) i dodanie poprzez: Znacznik<link>:

<link href="style.css" rel="stylesheet" type="text/css">

Umożliwia, to zwiększenie czytelności kodu. Szczególnie gdy konfigurujemy wygląd większej ilości elementów. Zapobiega to rozrastaniu się dokumentu do dużych wielkości, poprzez kolejne linijki kodu zawierające własności CSS. Ciężko, będzie dobrać się do treści, a sam dokument będzie mniej czytelny.

Możliwa jest również zmiana tła danego elementu np. poprzez użycie właściwości background. Kolor tła zapisujemy analogicznie jak to było w przypadku koloru czcionki.

Podsumowanie:

Style CSS:

  • Odpowiadają za prezentację treści strony
  • Nie mają jednej wersji (rozwój modułów), potocznie mówi się, że aktualna wersja to 3
  • Przeglądarki potrafią różnie interpretować CSS co prowadzi do tego, że…
  • … pisanie stylów powoduje wiele frustracji u programistów (ale jest też niezłym wyzwaniem :))
  • Całą specyfikację znajdziesz na stronie w3c: https://www.w3.org/TR/CSS/

CSS II

Style załączać do kodu HTML poprzez dodanie atrybutu style do znacznika:

<znacznik style="color: yellow;">treść</znacznik>

Styl przy znaczniku z wykorzystaniem atrybutu style nazywamy stylem inline (liniowym). W ten sposób dodane style są stosowane tylko do znacznika, przy którym występują.

Czy to ma sens?

Takie stylowanie wprowadza spore zamieszanie (bałagan) do kodu. Sprawia, że kod jest mniej czytelny, już przy niewielkiej ilości zdefiniowanych własności CSS przy jednym elemencie. Kod się przez to rozrasta i coraz ciężej się przez to w nim połapać.

Dlatego w CSS'ie istnieją różne typy selektorów, dzięki którym możemy łatwiej przypisać co jak ma być ostylowane. Rodzaje selektorów:

  • element - znacznik,
  • # - id,
  • class - klasa.

Dzięki selektorom możemy dane własności CSS przypisać do jednego konkretnego obiektu nawiązując do jago id - identyfikatora(#), do jakiejś większej grupy dzięki zastosowaniu klasy (class) oraz poprzez przypisania do elementu.

Już do tej pory korzystaliśmy z metody stylowania po elementach. Czyli używaliśmy nazwy danego elementu (znacznika, tag'u) i do tego dopisywaliśmy kolejne własności. Np.:

body {
background: #fefefe; }

Ta metoda nie pozwala nam na wyróżnianie tych tagów, ponieważ dla każdego stosuje taki sam styl.

Łatwiej jest wyróżnić nam część, grupę elementów poprzez zastosowanie stylowania po klasie (class). Możemy ją przypisać kilku na raz elementom i dzięki temu stworzyć grupę podobnie wyglądającą w naszym dokumencie. Klasę dodajemy jako atrybut do znacznika:

<znacznik class="name">treść</znacznik>
 .name {
właściwość: wartość; }

Czyli naszym wskazaniem dla kodu, że poprzez style CSS odnosimy się do danej klasy jest użycie kropki (.) przed nazwą klasy. Jak to zostało zrobione w powyższym przykładzie.

Nazwę klasy zapisujemy kebab-case'em

Stylowanie po id (#) umożliwia przypisanie konkretnych własności dla konkretnego box'u. Podobnie jak przy stosowania zakotwiczenia przy nawigacji w dokumencie, tak samo odwołujemy się do tego samego atrybutu poprzez selektor id. W tym przypadku w naszym kodzie poprzez zastosowanie znaczka # (hash) w pliku CSS'owym i nazwy identyfikatora, odnosimy się do tego elementu.

<znacznik id="name">treść</znacznik>
 #name {
właściwość: wartość; }

Jak wcześniej pisałem, że ważna jest zasada, która mówi, że powinno być jedno id na stronie.

Możliwe jest również łączenie selektorów. Daje to na przykład możliwość zmiany koloru odnośników (a) tylko w jednej sekcji (np. w nav - nawigacji dokumentu). Żeby to się udało musimy zapisać w pliku CSS:

nav a {
color: #04ac26; }

Taki zapis spowoduje, że własności zmienią się dla linków tylko w bloku nawigacji. Mówimy, że zagnieżdzamy style, czyli mówimy przeglądarce, że ostyluj tylko wybrany element znajdujący się w tym bloku.

Mamy również możliwość z skorzystania z !important. Wprowadza on sporo zamieszania w kod i jest zmorą jeśli chodzi o stylowanie. Nie powinno się go używać. Użycie go spowoduje, że będzie stosować daną własność elementu dla całego dokumentu. Nawet pomijając własności określone dla danego elementu dla danej sekcji.

nav a {
color: #04ac26; } a {
color: #c77d0f !important; }

Powyższy zapis spowoduje, że wszystkie linki będą miały jeden kolor mimo, że dla nawigacji mają przypisany inny kolor.

Zagadka 🙂 Jakiego koloru będzie treść?

#name {
color: red; } <span style=”color: yellow;” id="name">treść</span>

Odpowiedź: żółta 😛

Specyficzność w CSS

Jest to reguła ważności, która określa kiedy stosować jaki styl dla danego elementu. W szczególności gdy jest do niego przypisane kilka własności. Działa ona podobnie jak klasyfikacja medalowa na igrzyskach, że liczy się pierwsze złote medale przed srebrnymi i brązowymi. W naszym przypadku najważniejszy jest:

  1. !important
  2. style = "" - styl liniowy,
  3. #name - id,
  4. .name - klasa,
  5. element.

Dla przykładu mamy dwa przypadki w kilku przykładach:

  1. Wygrywa pierwszy, ponieważ więcej ma wymienionych elementów.

  2. Wygrywa dugi przypadek, ponieważ odwołuje się bezpośrednio do id, co jest wyżej w klasyfikacji.

  3. Tutaj również wygrywa drugi przypadek mimo, że oba odwołują się do id ale w drugim występuje odwołanie do klasy co jest wyżej cenione w klasyfikacji.

  4. Tutaj wygrywa drugi przypadek, w którym występuje odwołanie do !importanta

  5. Tutaj ważniejszy jest pierwszy przypadek, w którym jest przypadkiem liniowym (inline)

  6. Kolejny przypadek !importanta który wygrywa z wszystkimi (drugi przypadek)

Jeśli specyficzność jest równa to pierwszeństwo ma selektor "bliżej" elementu. Czyli jeśli style mamy załączone z pliku w sekcji head. To styl niżej znajdujący się będzie używany. Trzeba sobie wyobrazić, że przeklejamy plik ze stylami do dokumentu html i przeglądarka analizując kod zastosuje tą własność która ma mniejszą odległość między miejscem użycia, a samą własnością.

Warto dodawać klasę do wszystkich elementów i unikać zagnieżdżeń. Ułatwi to dalszą pracę i uniknie się konfliktów specyficzności. Zawszę będą brane pod uwagę te style, które zamierzamy użyć.

Nazewnictwo klas:

  • używaj pojedynczych słów z j. angielskiego,
  • nazywaj rzeczy zgodnie z ich funkcją na stronie,
  • startuj z małej litery np. header nie Header,
  • nie zaczynaj od cyfry,
  • jeśli musisz używać dwóch słów - użyj tzw. kebab-case np. new-listing,
  • staraj się, żeby element miał jedną klasę (nie zwiększaj specyficzności bez potrzeby.

Podsumowanie:

  • htlm class="name" -> css .name,
  • html id="name" -> css #name.
  • stylujemy po klasach (bez zagnieżdzeń)
  • nazywamy klasy po angielsku zgodnie z przeznaczeniem,
  • tworzymy klasy nawet, gdy mamy jeden element danego typu,
  • specyficzność może dać popalić 🙂
  • nie używamy !important i stylów inline'owych.

Bonusik :)

Życie w pudełku: Display, wymiary i overflow

Strony internetowe są dwuwymiarowe, co wiąże się z tym jak są one projektowane.

Poznajemy nowy znacznik:

<div>
nie ma znaczenia semantycznego, używamy gdy chcemy coś "opakować" (tylko kiedy nie istnieje odpowiedni semantyczny znacznik)

Nowe własności:

display: block
własność, która powoduje, że obiekt będzie wyświetlany jeden pod drugim i domyśle szerokość będzie miał ustawioną na cały ekran. Własność tą domyśle mają ustawione np article, main, znacznik p i section.
display: inline
własność, która powoduje, że obiekt wyświetlany będzie jeden obok drugiego w ten samej linii. Domyślny rozmiar obiektu jest taki jak wielkość zawartości (treści w obiekcie).
display: inline-block
własność, która łączy cechy block i inline. Sprawia, że obiekty zachowują się jak obiekty inline, czyli będą wyświetlać się obok siebie. Możliwe jest również nadanie im szerokości i wysokości (co nie jest możliwe w obiektach liniowych).
max-width /max-height
własność, która określa wielkość maksymalną kontenera. Czyli będzie się zwiększać wraz ze wzrostem okna, aż do momentu, kiedy osiągnie on wielkość maksymalną.
min-width / min-height
własność, która określa wielkość minimalną kontenera. Czyli będzie się on zmniejszać, aż do osiągnięcia wartości zadanej.
overflow: hidden
własność, która pozwala nam na odcinanie treści, która wystaje poza kontener (jeśli jest zadany max-height i tekstu jest więcej niż wielkość box'a). Treść, która została odcięta jest niedostępna (nie widzimy tego co znajduje się poza obszarem).
overflow: visible
domyślna dla overflow, która pozostawia zawartość kontenera widoczną i może ona wystawać poza kontener.
overflow: auto
wartość, która dodaje nam scroll w elemencie. Co sprawia, że zawartość nie wystaje poza kontener, a jednak jest widoczna po przewinięciu scroll'a

Życie w pudełku: Jednoski

Jednostki zapisujemy razem z liczbą (np. 10px). W innym wypadku przeglądarka, nie będzie mogła zrozumieć o jaką jednostkę chodzi i styl nie zostania załadowany. Podstawowe jednostki:

  • Zero w CSS'ie jest zawsze zerem i możemy zapisywać bez jednostek.
  • piksele (px) - określają jaką wielkość ma mieć box w pikselach
  • procenty (%) - określają jaką wielkość ma mieć box w stosunku do okna przeglądarki lub rodzica w którym się obiekt znajduje jeśli jego wielkość jest określona,
  • viewport width (vw) - określa jaką wielkość (szerokość) procentową ma mieć box do wielkości okna w przeglądarce,
  • viewport height (vh) - określa jaką wielkość (wysokość) procentową ma mieć box

Życie w pudełku: BOX MODEL (padding/ margin/ border)

Jest fundament budowy strony internetowej (layout'u). Zakłada on, że wszystkie elementy otoczone są "niewidzialnym" prostokątem.

Z czego składa się nasze pudełko:

padding
margines wewnętrzny - jest częścią elementu (czyli jego rozmiar jest doliczany do rozmiaru elementu). Określa o ile elementy wewnątrz boxu (np. tekst) będą oddalone od krawędzi tego kontenera. Mamy kilka rodzajów zapisu paddingu:
  • Zapis pełny: 
                                            
                                            padding-top: 3px;
                                            padding-right: 4px;
                                            padding-bottom: 5px;
                                            padding-left: 6px;
  • Zapis skrótowy (shorthand) 
                                            
                                            padding: 3px 4px 5px 6px;
                                            gdzie:
                                            padding: top right bottom left;
    Żeby zapisać osobno kolejne wartości paddingu dla góry, prawej strony, dołu, lewej strony to zaczynamy od góry przesuwamy się zgodnie z ruchem wskazówek zegara, czyli następna będzie prawa strona, dół i lewa strona. Możemy w ten sposób określić różne odstępy dla każdego boku pudełka. Zasada ta działa to każdej własności CSS'owej, która ma kierunki.
  • wartości lustrzane - pozwala zapisać padding w dwóch wartościach. Pierwsza będzie się odnosić do odstępu od góry i dołu, a druga będzie dotyczyła lewej i prawej strony pudełka. Ten zapis sprawi, że po jego użyciu odstęp będzie taki sam z góry i z dołu oraz taki sam dla prawej lewej strony elementu. Zasada ta działa to każdej własności CSS'owej, która ma kierunki.
    padding: 3px 4px;
  • wartości wszystkie takie same - przy tym zapisie odstęp z każdej strony jest taki sam i użyta jest tylko jedna wartość. Zasada ta działa to każdej własności CSS'owej, która ma kierunki.
    padding: 16px;
border
ramka elementu - podobnie jak padding jest częścią elementu. Jest to ramka która otacza element. Inne własności dla border:
  • border-width - określa grubość ramki. Występuje dla niego zapis pełny, skrócony, lustrzany i wszystkie wartości takie same.
  • border-style - określa styl linii ramki (może być ciągła, przerywana, kropkowana itd.)
    • solid - linia ciągła,
    • dashed - linia kreskowana,
    • dotted - linia kropkowana.
  • border-color - określa kolor ramki,
Istnieje skrócony sposób zapisu, który umożliwia zapisanie wszystkich tych własności w jednej linijce:
border: width style color;
  • border-radius - zaokrągla nasz element. Podajemy wielkość promienia, jaki ma być zastosowany do zaokrąglenia rogów elementu. Możemy zrobić koło z naszego elementu przy wartości border-radius: 50%. Trzeba tylko pamiętać, że treść w rogach będzie przycięta. Można temu zaradzić stosując odpowiedni padding.
margin
margines zewnętrzny - nie jest częścią elementu (czyli jego wielkość, nie zwiększa rozmiaru elementu. Odpowiada za to o ile możemy się odsunąć od innych elementów. Wartości zapisujemy tak samo jak dla paddingu.
  • marin: 10px auto; - sprawi, że przeglądarka z prawej i lewej strony dobierze taki sam margines, czyli element się wyśrodkuje (będzie to działać dla elementów blokowych). Żeby ta własność zadziałała, element musi mieć zdefiniowaną szerokość. w innym przypadku cały czas będzie domyślnie szeroki na całe okno, czyli nie będzie miał się z czego środkować. Nie jest to prawidłowy sposób wyśrodkowywania elementów.

Życie w pudełku: box-sizing

Box-sizing to wielkość naszego pudełka (elementu). Wielkość pudełka możemy mierzyć na dwa sposoby:

content-box
czyli bierzemy pod uwagę samą wielkość zawartości elementu, bez uwzględnienia paddingu, ramki i marginesu zewnętrznego. Czyli Te wartości dodatkowo trzeba będzie w pamięci dodawać do wielkości elementu.
border-box
w tym przypadku bierzemy pod uwagę wielkość elementu razem z ramką i paddingiem. Czyli wielkość zawartości zostanie automatycznie dobrana po odliczeniu borderu i paddingu.

Życie w pudełku: position

Własność position umożliwia nam pozycjonowanie elementów w oknie przeglądarki. Nie musimy dzięki niej korzystać z zdefiniowanych w przeglądarce układów. Możemy określić, że ten element ma znajdować się dokładnie w tym i tym miejscu.

Wartości możliwe do zdefiniowania dla position:

static
wartość domyślna. Każdy kolejny element jest umieszczany poniżej elementu poprzedniego.
relative
pozycjonowanie względne - jej położenie jest zależne od elementów otaczających i względem nich jest przesuwany (umożliwia na przesunięcie elementu względem miejsca, w którym zostałby umieszczony przy wartości static).
absolute
pozycjonowanie bezwzględne, czyli określa dokładne położenie danego elementu w oknie przeglądarki i nie wpływa on na położenie innych elementów (może się zdarzyć, że będzie nachodził na inne elementy). Możliwe jest również określenie dokładnej pozycji w elemencie, gdy ten element będzie miał własność position: relative;
fixed
ma bardzo podobne własności jak absolute. Różnica polega na tym, że pozycjonowanie elementu następuje względem okna przeglądarki czyli dany element będzie zawsze wyświetlany w tym samym miejscu (kiedy zawartość będzie przesuwana, element nie będzie się przesuwał. Będzie wyświetlany w tym samym miejscu).
sticky
sprawia, że dany element będzie odpowiednio pozycjonowany w rodzicu. Gdy element dojedzie do górnej krawędzi swojego rodzica, zatrzyma się przy górnej krawędzi (lub odsunięty od niej jeśli odpowiednio zdefiniujemy top) i będzie wyświetlany i nachodził na elementy niżej, aż do końca swojego rodzica i wtedy zniknie razem z rodzicem gdy jego zawartość będzie przesuwana (przeglądana). Żeby funkcja działa rodzic nie może mieć overflow'a i trzeba określić własność top (odsunięcie od górnej krawędzi).

Nowe własności:

top, left, right, buttom
własności przesunięcia, określa odległości odstępu od krawędzi danego pudełka.
z-index
pozwala na zdefiniowanie kolejności wyświetlania elementów. Gdy elementy nachodzą na siebie, ta własność pozwala nam na zdefiniowanie, który element ma przykryć inne elementy. Wartościami własności są liczby. Im wyższa ona będzie tym wyżej będzie wyświetlany element (czyli 150 będzie wyświetlane nad 100 itd.).
