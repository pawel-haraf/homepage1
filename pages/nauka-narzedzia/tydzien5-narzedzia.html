Debugging

Czyli co zrobić jeśli w projekcie coś nie działa?

  1. Sprawdź czy plik jest zapisany?
  2. Sprawdź czy edytujesz właściwy plik
  3. Sprawdź czy odświeżyłeś przeglądarkę - często pamięć cache powoduje, że strona nie odświeża się jak powinna. Ponieważ część danych ładuje do pamięci cache (grafiki, fonty) i nie za każdym razem te pliki odświeża (dzięki temu strony szybciej się ładują) . Możemy to rozwiązać poprzez zaznaczenie w zakładce network devTools'ów opcję Disable cache. Co spowoduje, że wyłączymy pamięć cache. Można również odświeżyć stronę poprzez 'ctrl+f5'. Jest ot tzw. twarde odświeżenie strony.
  4. Sprawdź czy nie masz błędów w edytorze/devtoolsach
  5. Czytaj błędy. W komunikatach jest zawarta jest informacja co jest nie tak (przynajmniej wskazanie w jakim miejscu powinniśmy szukać błędu.
  6. Zajrzyj w zakładkę network, często są tam informacje, dlaczego dany plik się nie ładuje (wyświetlany jest tam statusy wszystkich plików i może być np błąd 404,  czyli pod daną lokalizacją nie danego pliku i przeglądarka ie wie gdzie on się znajduje -  może być zła nazwa pliku wpisana ).
  7. Pogadaj z kaczką - często opisanie kodu komuś (np. gumowej kaczce) pomaga w znalezieniu błędu w kodzie.
  8. Google it (wygoogluj) sprawdź czy rozwiązanie jakiegoś błędu nie jest opisane gdzieś w internecie.
    1. Nie chodzimy na w3schools - znajduje się tam dużo starych i niepoprawnych technik w3c != w3schools
    2. Stack Overflow
    3. Inne skoko żródła:
      1. Dokumentacja narzędzia z którego korzystasz
      2. GitHub issues
      3. Dokumentacja MDN
      4. DevDocs API Documentation
      5. Can I use
      6. CSS Almanac
      7. HTML 5.2
      8. CSS
  9. Fajne grupy polskojęzyczne:
    1. HTML, CSS i JS: pierwsze kroki
    2. Front-end: pierwsza praca w HTML5, CSS3 i JavaScript
    3. Programowanie - wsparcie na starcie
    4. Programuj dziewczyno
    5. Grupy lokalnych spotkań programistycznych (meet.js, dev.js)

Figma - grafik cięcie, też zajęcie

W tej lekcji będziemy tworzyć prostą stronkę, która będzie zawierać naszą wizytówkę. Do stworzenia projektu graficznego będziemy używać programu Figma .

  1. Tworzymy nowe repozytorium business-card na naszym koncie na GitHubie  i poleceniem git clone adres-naszego-repozytorium ściągamy je do katalogu dev. Zostanie w katalogu dev utworzony nowy katalog, w którym będzie nasze repozytorium z wizytówką
  2. Uruchamiamy VSC w katalogu z naszą wizytówką i tworzymy plik index.html z podstawową strukturą dokumentu HTML, oraz style css (pliki style.css i normalize.css).
  3. Przystępujemy do przenoszenia elementów projektu na html
  4. Aby ustawić odpowiednie własności czcionki w wizytówce wykorzystujemy:
    1. font-familyrodzaj czcionki
    2. font-size wielkość czcionki
    3. font-weight grubość czcionki
    4. font-style styl czcionki
    5. line-height wysokość linii
    6. text-decoration dekoracje tekstu (podkreślenia)
  5. Wykonana wizytówka znajduje się pod adresem: href https://pawel-haraf.github.io/business-card-2020/.
