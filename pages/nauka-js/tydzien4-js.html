Typy danych i immutability

Dla przypomnienia: zmienna to takie połączenie z naszym wszechświatem wartości.  Zmienna to nie wartość!

Zmienne w JS 
const
zmienne mające stałe połączenie z jedną wartością (tego połączenia nie możemy zmienić):
const name = 'Maciek';
const age = 31;
let
zmienne, których połączenie możemy przepinać, ale nazwę zmiennej definiujemy tylko raz!
let isHappy = false;
isHappy = true;
isHappy = 'tak';
var
na razie nie będziemy jej stosować o niej dowiemy się później.

Do nazywania zmiennych używamy camelCase.

Typy danych

Zmienne mogą być połączone do różnych typów prostych (rodzajów danych). Typy proste (primitives) w JS:

  • String - tekst, otaczamy ' lub " (zalecam ')
  • Number - liczba
  • Bigint - duża liczba ;) większa niż 253
  • boolean - true lub false
  • null - wartość celowo niezdefiniowana
  • undefined - zmienna jeszcze nie zdefiniowana (jeszcze nie istnieje)
  • *symbol - tworzy "unikalne opakowanie"

Żeby sprawdzić jakie typu jest zmienna używamy polecenia:

typeof nazwaZmiennej

Jeśli zrobimy przypisanie zmiennej do zmiennej np.:

let x=10;
let y=10;

To uzyskamy połączenie do wartości pod którą podpięte było x czyli pod zmienną y będzie podpięta 10. To oznacza, że połączenie jest pod wartość zmiennej, a nie pod samą zmienną. Gdy zmienimy podpięcie zmiennej x (x=20), to podpięcie zmiennej y nie ulegnie zmianie (dalej będzie do wartości 10).

IMMUTABILITY (niezmienność, niemutowalność)

Czyli nie możemy zmieniać wartości primitive values w JS (one już istnieją, a my jedynie się do nich łączymy). To co zmieniamy to połączenie między nimi, a zmiennymi.

DOM'owe przedszkole

DOM to skrót od Document Object Model (czyli obiektowy model naszego dokumentu).  Dom to jest to co widzimy w Devtools'ach (nie jest to HTML, ani źródło strony).

DOM jest ostateczną wersją naszego dokumentu, która wyświetla się w przeglądarce (dlatego to co widzimy w devtool'ach to jest DOM). HTML to tylko instrukcja dla przeglądarki jak stworzyć stronę, a DOM to już gotowa strona.

JavaScript jest taką warstwą, która lubi bardzo DOM. Pozwala on komunikowanie się (chodzenie w interakcje) własnie z zawartością strony poprzez javaScript. Możemy się dostawać do jej elementów i je modyfikować.

Żeby tego dokonać musimy się podpiąć do naszego świata wartości (do DOM'u) i tu dopiero podpinać się możemy pod pewne elementy. Do tego celu wykorzystujemy polecenie:

document.querySelector('.selector');

W naszych skryptach wpisujemy:

const heading = document.querySelector ('.main__header');

console.log(heading)

Co daje nam odpowiedź w konsoli w przeglądarce:

<h1 class="main__header" id="name">Paweł Haraf - Strona domowa</h1>

Żeby nie wprowadzać większego zamieszania i uniknąć problemów przy późniejszej zmianie nazwy klasy dobrą praktyką jest dodawanie dodatkowej klasy za pomocą BEM'a:

block__element--js

Dzięki temu możemy modyfikować treść elementu znajdującego się w danym obiekcie. Dokonujemy za pomocą funkcji:

nazwaZmiennejQuerySelector.innerHTML = 'treść'

Czyli w naszym przypadku zapisujemy:

const heading = document.querySelector ('.main__header--js');

console.log(heading)

heading.innerHTML = "Witaj świecie pisze do Ciebie za pośrednictwem JavaScript"
Podsumowanie
  • DOM to "złożony" w przeglądarce HTML
  • JavaScript może modyfikować DOM
  • funkcja querySelector zwraca node (węzeł) z HTML-a, selektor jest zapisywany zgodnie z CSS
  • korzystając z BEM - tworzymy dodatkową klasę z modyfikatorem --js (ułatwia identyfikację / usuwa potencjalne problemy)
